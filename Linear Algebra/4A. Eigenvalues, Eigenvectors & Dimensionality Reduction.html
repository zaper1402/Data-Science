1. Eigenvalues & Eigenvectors¶

1.1 Definition¶

For a square matrix $A$ of shape $(n \times n)$ and a non-zero vector $\mathbf{v}$ of size $n$, if:

$$ A \mathbf{v} = \lambda \mathbf{v}, $$

then:

  • $\lambda$ is called an eigenvalue of $A$.
  • $\mathbf{v}$ is the corresponding eigenvector.

1.2 Why Important in Data Science?¶

  • Principal Component Analysis (PCA) uses the eigenvectors/eigenvalues of the covariance matrix to find directions of maximum variance in data.
  • Stability: In dynamical systems or Markov chains, eigenvalues describe equilibrium and long-term behavior.
  • Dimensionality Reduction: The eigenvectors can represent “principal directions”; largest eigenvalues correspond to largest variance directions.

1.4 Visualizing the Transformation¶

A 2D transformation can be visualized by applying $A$ to a grid of points, but let’s keep it simple here. We'll:

  1. Plot a set of vectors in 2D.
  2. Multiply by $A$.
  3. Show how eigenvectors are scaled (not rotated) when multiplied by $A$.
In [4]:
import numpy as np
import matplotlib.pyplot as plt

# Define a 2D transformation matrix
A = np.array([[2, 1],
              [1, 3]])

# 1. Create a grid of points
N = 11  # how many points in each axis
x_vals = np.arange(-1.5, 2, 0.5)
y_vals = np.arange(-1.5, 2, 0.5)

X, Y = np.meshgrid(x_vals, y_vals)

# Stack all points into an (N*N, 2) array for matrix multiplication
points = np.stack([X.ravel(), Y.ravel()], axis=1)  # shape: (N*N, 2)

# 2. Transform the points by A
transformed_points = points @ A.T  # shape: (N*N, 2)

# 3. Compute eigenvalues & eigenvectors
eigvals, eigvecs = np.linalg.eig(A)

# Plot the results side-by-side
fig, axes = plt.subplots(1, 2, figsize=(12, 6))
axes[0].scatter(points[:, 0], points[:, 1], s=20, alpha=0.7, color='blue')
axes[0].axhline(0, color='black', linewidth=1)
axes[0].axvline(0, color='black', linewidth=1)
axes[0].set_title("Original Grid", fontsize=18)
axes[0].set_aspect('equal', 'box')

# Plot eigenvectors (from origin) on the original grid
for i in range(len(eigvals)):
    vec = eigvecs[:, i]
    # We'll draw a simple arrow indicating the direction of the eigenvector
    axes[0].arrow(0, 0, vec[0] * 1.5, vec[1] * 1.5, head_width=0.1, color='green', lw=2, length_includes_head=True)
    axes[0].text(vec[0]*1.6, vec[1]*1.6, f"v{i+1}", fontsize=16, color='green')

axes[1].scatter(transformed_points[:, 0], transformed_points[:, 1], s=20, alpha=0.7, color='red')
axes[1].axhline(0, color='black', linewidth=1)
axes[1].axvline(0, color='black', linewidth=1)
axes[1].set_title("Transformed by A", fontsize=18)
axes[1].set_aspect('equal', 'box')

# Show how eigenvectors get scaled by eigenvalues in the transformed space
# For each eigenvector v, A * v = lambda * v (so it should end up at (lambda*v))
for i in range(len(eigvals)):
    lambda_i = eigvals[i]
    vec = eigvecs[:, i]
    transformed_vec = A @ vec  # which should be same as lambda_i * vec
    # transformed_vec = lambda_i*vec
    
    axes[1].arrow(0, 0, transformed_vec[0], transformed_vec[1], 
                  head_width=0.1, color='g', lw=2,
                  length_includes_head=True)
    axes[1].text(transformed_vec[0]*1.1, transformed_vec[1]*1.1, f"λ{i+1}v{i+1}", fontsize=14, color='g')

plt.tight_layout()
plt.show()

2. Principal Component Analysis (PCA)¶

Applications of Principal Component Analysis (PCA)¶

  1. Feature Reduction & Noise Filtering

    • High-dimensional datasets (e.g., genomics, image data, text embeddings) can be projected onto fewer dimensions.
    • Reduces storage and computational costs; also helps visualize data in 2D/3D.

  2. Image Compression

    • Using PCA (or SVD) to represent images with a low-rank approximation.
    • Fewer components → less data needed → compression without losing too much quality.

  3. Denoising

    • In signals or images, principal components often capture the main structure; smaller components capture noise.
    • Truncating those smaller eigenvalues effectively filters out noise.
In [ ]:

Why Dimensionality Reduction Matters¶

  • Efficiency: Fewer features → faster algorithms (like classification, clustering, regression).
  • Interpretability: Visualizing data in 2D/3D to identify patterns or cluster separation.
  • Better Generalization: Removing noisy or redundant features often improves model performance (less risk of overfitting).

2.1 Conceptual Overview¶

  • Goal: Reduce high-dimensional data to fewer dimensions while retaining most variance.
  • Steps:
    1. Compute the mean of the data (centering).
    2. Form the covariance matrix of the (centered) data.
    3. Eigen-decompose the covariance matrix to find principal components (the eigenvectors).
    4. Sort components by descending eigenvalues (explained variance).
    5. Project the data onto the top principal components.

2.2 Why Eigen-Decomposition?¶

  • The covariance matrix is symmetric and real, so it can be decomposed into: $$ \text{Cov}(X) = V \Lambda V^T, $$ where $\Lambda$ is a diagonal matrix of eigenvalues (variances along principal axes), and $V$ are the eigenvectors (principal directions).

2.3.2 Step-by-Step PCA¶

  1. Center the data by subtracting the mean.
  2. Compute the covariance matrix.
  3. Eigen-decompose the covariance matrix.
  4. Sort the eigenvalues and eigenvectors (largest eigenvalue => first principal component).
  5. Project data onto top principal components.

Let's do this manually.

  • The sum of the eigenvalues = sum of all variances across features.

3. Key Takeaways¶

  1. Eigenvalues & Eigenvectors:

    • Found by solving $A\mathbf{v} = \lambda \mathbf{v}$.
    • Describe how a matrix (linear transformation) scales/directionally transforms vectors.
    • Crucial in many ML algorithms (e.g., PCA, spectral clustering, stability analysis).

  2. PCA:

    • Uses the eigen-decomposition of the covariance matrix to identify directions of maximum variance.
    • Allows us to reduce dimensionality while preserving as much variation in the data as possible.
    • The eigenvalue magnitudes indicate how much variance each principal component captures.

  3. Practical Tips:

    • In real-world datasets, we often use scikit-learn’s PCA for convenience and numerical robustness.
    • Be mindful of center/scale: typically center data before PCA, sometimes also scale features if they have very different ranges.
    • Eigen-decomposition is more direct for smaller datasets; for large datasets, SVD-based approaches are usually preferred for efficiency and stability.

With these concepts, you have a strong basis for understanding how eigenvalues, eigenvectors, and PCA power many dimensionality reduction and feature extraction techniques in data science.
