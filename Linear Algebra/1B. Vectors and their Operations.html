1. What is a Vector?¶

A vector is a mathematical object that can be:

  • Geometrically interpreted as an arrow in space (e.g., 2D plane, 3D space).
  • Algebraically represented as an ordered list of numbers (called components).
In [3]:
# Creating a 2D vector and a 3D vector
v2 = np.array([3, 4])
v3 = np.array([1, 2, 3])

print("2D vector:", v2, "Shape:", v2.shape)
print("3D vector:", v3, "Shape:", v3.shape)

# Note: np.array([3,4]) is basically 1-dimensional in Python with length 2.

2D vector: [3 4] Shape: (2,)
3D vector: [1 2 3] Shape: (3,)

1.1 Geometric Interpretation¶

  • In 2D, a vector could look like an arrow drawn from the origin (0,0) to a point (x, y).
  • In 3D, it would be an arrow from (0,0,0) to (x, y, z).
  • You can visualize it as describing both magnitude (length) and direction.

1.2 Algebraic Representation¶

A vector can be written as a column of numbers (most common in math) or a row of numbers in many programming contexts. For instance, a 2D vector might be:

$$\begin{pmatrix} x \\ y \end{pmatrix}\quad \text{or simply as an array } [x, \, y].$$

1.3 Notation & Dimensions¶

  • A vector in $\mathbb{R}^n$ is an ordered list of $n$ real numbers.
  • Example in 3D: $\mathbf{v} = (v_1, v_2, v_3)$.
  • The dimension of the vector is the number of components it has.

2. Scalar vs. Vector vs. Matrix¶

  • Scalar: A single number (e.g., 7, 3.14). In Python, just a float or int.
  • Vector: A 1D array of numbers (e.g., [3, 5, 2]).
  • Matrix: A 2D grid of numbers (e.g., a table of shape $m \times n$).

In [ ]:

Typical Uses in Data Science¶

  • Scalars: learning rate in gradient descent, a single measurement, etc.
  • Vectors: a data point with multiple features, model parameters in linear regression.
  • Matrices: entire datasets (rows = samples, columns = features), weight matrices in neural networks.

3. Common Vector Operations¶

Let's look at the most important operations you'll do with vectors.

3.1 Vector Addition and Subtraction¶

  • Addition: If $\mathbf{u} = (u_1, u_2, ..., u_n)$ and $\mathbf{v} = (v_1, v_2, ..., v_n)$, then $$\mathbf{u} + \mathbf{v} = (u_1 + v_1, \, u_2 + v_2, \, ..., u_n + v_n).$$
  • Subtraction: $\mathbf{u} - \mathbf{v} = (u_1 - v_1, \, u_2 - v_2, \, ..., u_n - v_n).$
In [6]:

# Addition

print("u + v =", add_uv)

u + v = [4 7 6]

# Subtraction

Geometrically, this corresponds to placing the tail of $\mathbf{v}$ at the head of $\mathbf{u}$ and drawing the diagonal of the parallelogram formed.

# Let's define two 2D vectors u and v
u = np.array([3, 2])
v = np.array([2, 4])

# For convenience in plotting arrows from the origin, we'll define:
origin = np.array([0, 0])  # coordinate (0,0)

# Vector addition
u_plus_v = u + v

plt.figure(figsize=(5,5))

# Plot vector u (blue)
plt.quiver(*origin, *u, color='blue', angles='xy', scale_units='xy', scale=1, label='u')

# Plot vector v from the HEAD of u (so the tail is at u)
# To do this, we shift the origin to u
plt.quiver(*u, *(v), color='red', angles='xy', scale_units='xy', scale=1, label='v (shifted)')

# Plot the resultant (u+v) from the origin
plt.quiver(*origin, *u_plus_v, color='green', angles='xy', scale_units='xy', scale=1, label='u+v')

# Formatting the axes
plt.xlim(-1, 7)
plt.ylim(-1, 7)
plt.axhline(0, color='grey', linewidth=0.5)
plt.axvline(0, color='grey', linewidth=0.5)
plt.grid()
plt.title("Vector Addition: u + v")
plt.legend(loc=2)
plt.show()

Notes:

  • The blue arrow is $\mathbf{u}$.
  • The red arrow starts at the tip of $\mathbf{u}$ and represents $\mathbf{v}$.
  • The green arrow is the resultant $\mathbf{u} + \mathbf{v}$ starting from the origin.

3.2 Scalar Multiplication¶

  • If $c$ is a scalar and $\mathbf{v} = (v_1, v_2, ..., v_n)$, then: $$c \mathbf{v} = (c \cdot v_1, \, c \cdot v_2, \, ..., c \cdot v_n).$$
  • Geometrically, scalar multiplication stretches (if $|c|>1$) or shrinks (if $|c|<1$) the vector length, and if $c < 0$, it flips the direction.

3.3 Dot Product¶

  • The dot product of two vectors $\mathbf{u}$ and $\mathbf{v}$ (of the same dimension) is defined as: $$\mathbf{u} \cdot \mathbf{v} = \sum_{i=1}^{n} u_i \, v_i.$$ In 2D, that looks like: $$(u_1, u_2) \cdot (v_1, v_2) = u_1 v_1 + u_2 v_2.$$

  • Geometric Interpretation: $$\mathbf{u} \cdot \mathbf{v} = \|\mathbf{u}\| \|\mathbf{v}\| \cos(\theta),$$ where $\theta$ is the angle between the two vectors, and $\|\mathbf{u}\|$ is the magnitude (length) of $\mathbf{u}$.

    • If the dot product is positive: vectors point more or less in the same direction (angle < 90°).
    • If the dot product is zero: vectors are perpendicular (angle = 90°).
    • If the dot product is negative: vectors point more or less in opposite directions (angle > 90°).

  • Projection: The dot product can be used to project one vector onto another: $$\text{proj}_{\mathbf{v}}(\mathbf{u}) = \left(\frac{\mathbf{u} \cdot \mathbf{v}}{\|\mathbf{v}\|^2}\right) \mathbf{v}.$$

# Let's compute the angle between u and v
u_norm = np.linalg.norm(u)
v_norm = np.linalg.norm(v)
cos_theta = dot_value / (u_norm * v_norm)

theta_radians = np.arccos(cos_theta)
theta_degrees = np.degrees(theta_radians)

print("||u|| =", u_norm)
print("||v|| =", v_norm)
print("cos(theta) =", cos_theta)
print("Angle between u and v (in degrees) =", theta_degrees)

||u|| = 3.605551275463989
||v|| = 4.123105625617661
cos(theta) = 0.7399400733959438
Angle between u and v (in degrees) = 42.27368900609373
  • We used np.linalg.norm(u) to get the magnitude of vector u.
  • cos_theta = (u · v) / (||u|| * ||v||).
  • theta_radians = arccos(cos_theta).
  • In practice, if cos_theta is close to 1, the angle is small (vectors are similar in direction). If it's near 0, they're perpendicular.

3.5. Visualizing Vector Norms¶

Concept: The norm (or length) of a vector $\mathbf{u}$ is $\|\mathbf{u}\| = \sqrt{u_1^2 + u_2^2 + \cdots}$. In 2D, that’s the distance from $(0,0)$ to $(u_1,u_2)$.

3.6. Visualizing Cosine Similarity¶

Concept: The cosine similarity between two vectors is: $$\text{cosine\_similarity}(u, v) = \frac{u \cdot v}{\|u\|\|v\|} = \cos(\theta).$$

  • Values range from -1 (opposite directions) to +1 (same direction).
  • If $\mathbf{u}$ and $\mathbf{v}$ were in similar directions, you’d expect a positive and close to 1 value.
  • In many ML tasks (e.g., text embeddings), a higher cosine similarity means more similar vectors.
In [17]:
cos_sim = np.dot(u, v) / (np.linalg.norm(u)*np.linalg.norm(v))
print("Cosine similarity between u and v =", cos_sim)

Cosine similarity between u and v = 0.7399400733959438
